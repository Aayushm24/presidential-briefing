## What changed from pipeline to published (key edits)

### 1. Hook rewritten — two-line binary instead of personal observer
**Pipeline:** "Every team i talk to says they deployed Claude Code. / Almost none built the infrastructure to compound it."
**Aayush:** "Everyone has Claude Code. Almost no one is using it right."

The personal observer hook is TIER 2. The two-line binary is sharper — universal claim on line 1, immediate inversion on line 2. Works better when the gap IS the point.

### 2. Bridge sentence added after hook
"I see it across my network, teams shipping with identical tools, but getting wildly different outcomes." and "The difference lies in what happens when your Claude conversation ends."

These two sentences ground the claim in his vantage point, then name the mechanism. Pipeline jumped straight from hook to case study.

### 3. Sequence fragments broken one-per-line
Pipeline stacked "Every pattern analyzed. Every skill saved to a shared repository. Every engineer, designer, and PM in the same system." as one block.
Aayush broke each into its own line. More breathing room, more rhythm.

### 4. Parenthetical personality aside added
"(using claude code is not a moat 🤷🏻‍♂️)" — inline, mid-sentence. Pre-empts the obvious objection with a wink.

### 5. Closing question made specific and present-tense
**Pipeline:** "what does your team's AI memory look like after 12 months of compounding?" — aspirational, 12 months out.
**Aayush:** "What does your team remember from last sprint's Claude Code usage?" — answerable today.

### 6. p.s. grounded in a new concrete detail, not a stat repeat
Pipeline p.s. repeated a percentage already implied in the body.
Aayush's p.s. landed a new specific detail (designers and PMs merging PRs) that made the stat mean something.

### 7. p.p.s. with Atlan context added
"at Atlan we've been building this layer for a while. happy to compare notes with anyone doing the same." — soft invitation, not a pitch. Grounds the post in real work.
