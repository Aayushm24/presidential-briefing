# Anti-AI-Slop Checklist (Posts)

Same checklist as briefs — see `.claude/skills/write-briefing/references/anti-slop-checklist.md` for the canonical version.

The Substitution Test, Specificity Count, Jargon Scan, Stat-Stat-Reframe-Metaphor Check, and "So What?" Test all apply to LinkedIn posts.

**Post-specific emphasis:**

1. **"Tell the story, then make the point"** — the concrete moment first, insight after. But only when the story is REAL (see aayush-experiences.md). If no real story exists, write a take — never fabricate.

2. **"Name names"** — every post needs at least one named company/person/tool. Banned: *"a well-known company"*, *"several teams"*, *"many founders"*.

3. **Insight-first, CTA last** — posts that lead with an ask are consistently bottom performers. Save the CTA for the last line.

4. **3 short sentences in a row** — sounds like a list, not a person thinking. Vary sentence length constantly.

5. **Same word twice in one post** — hard ban (other than conjunctions + proper nouns).

6. **Read-aloud test** — read the post out loud. If any sentence sounds like something an LLM would write but a person wouldn't say — rewrite.
